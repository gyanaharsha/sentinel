How the check stays private

Sending a password to a "checker site" would itself be a leak. Sentinel uses k-anonymity — the range-query protocol password managers use under the hood.

This isn't just my opinion

NIST's Digital Identity Guidelines (SP 800-63B, Revision 4 — the current US federal standard, finalized 2025) rewrote the password rulebook. The short version:

15+

Minimum length when a password is the only factor. Length beats everything — each character multiplies the search space.

No rules

Composition requirements ("must include a symbol") are now prohibited — "shall not" — because they push people into predictable patterns.

Blocklists

New passwords must be screened against breached and common passwords. That screening is literally what this page does.

No expiry

Forced periodic changes are out. Change a password when there's evidence of compromise — like it appearing above.

Source: NIST SP 800-63B-4, Sections 3.1.1–3.1.2
